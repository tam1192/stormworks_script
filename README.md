# gitの使い方兼こういう風にしてほしい
　こんな感じに使うといいと思います。

### 初めに
　ローカルリポジトリを複製します。 `git clone ～`をやってください。

### ファイルを変更する前に
　issueのページで・目的（バグ修正など）　・やりたいこと（簡単に　○○箇所のバグを修正） ・目標（結果どうなるか　）をまとめておきましょう。
また、似たようなissueがある場合はそちらに譲りましょう。（競合を起こさないために）

### ブランチを切る
　issueでやることを決めたら、ブランチを切ります。
